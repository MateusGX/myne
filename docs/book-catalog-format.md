# Book Catalog & Reading Log Format

This document describes the on-device storage format Myne uses to catalog physical books and track
reading sessions. All of this lives on the SD card under `/.myne/`. It's useful background if you're
working on `lib/DataStore/`, `src/network/MyneWebServer.cpp`, or the dashboard's data layer
(`dashboard/src/lib/api.ts`).

Everything is plain JSON / NDJSON (newline-delimited JSON) plus a handful of small binary files used as
caches. The binary files are **derived data** — they can always be regenerated from the JSON sources in
`books/` and `readings/`, which are the source of truth.

## Directory layout

```text
.myne/
├── books/                # one JSON file per book: {id}.json
├── catalog/              # generated index — safe to delete, rebuilt automatically
│   ├── idx.bin           # 27 x uint16 letter counts (A-Z, #)
│   ├── A.ndjson … Z.ndjson, #.ndjson
│   └── c/{id8}.ndjson     # per-collection book lists
├── collections.ndjson    # persistent collection id <-> name registry
├── notes/
│   ├── books/{bookId}.note       # per-book note (plain text)
│   └── collections/{id8}.note    # per-collection note (plain text)
├── readings/             # one JSON file per book: {bookId}.json (array of readings)
├── readings-sum/{bookId}.bin   # 17-byte per-book reading summary (cache)
├── stats-cache/
│   ├── summary.bin       # 40-byte global stats summary (cache)
│   ├── {year}.bin        # 24-byte per-year monthly session counts (cache)
│   └── {year}-{mm}.bin   # 62-byte per-month daily session counts (cache)
└── sync_needed           # empty flag file — present when catalog/ needs rebuilding
```

## Source of truth: `books/` and `readings/`

### `books/{id}.json` — one file per book

```json
{
  "id": "<generated id>",
  "t": "title",
  "a": "author",
  "c": "collection name (empty if standalone)",
  "v": "volume / number",
  "l": "location / shelf"
}
```

All fields except `id` are optional and default to `""`. `id` is generated by
`DataStoreUtils::generateId()` (lib/DataStore/) when the book is created and never changes.

### `notes/books/{bookId}.note` — optional note for a book

A per-book note, stored as **plain UTF-8 text** (not JSON). A missing or empty file means no note is
set. The web API exposes this as the singular `note` field.

### `readings/{bookId}.json` — one file per book, array of reading entries

```json
[
  {
    "id": "<reading id>",
    "s": "want | reading | paused | finished | dropped",
    "rt": 0,
    "sessions": [
      { "d": "2026-06-05", "tm": "21:30", "p": 42 }
    ]
  }
]
```

- `id` — generated by `ReadingLog::newId()`
- `s` — status, one of `want` (Want to Read), `reading`, `paused`, `finished`, `dropped`; defaults to
  `reading` if omitted
- `rt` — reading type: `0` = Page, `1` = Chapter; omitted means `0`
- `sessions` — chronological list of logged sessions, each with:
  - `d` — date as `YYYY-MM-DD` (empty if the device clock wasn't synced when logged)
  - `tm` — time as `HH:MM` (omitted if not recorded)
  - `p` — position (page or chapter number) at that session

A book can have multiple reading entries (e.g. separate "Reading" and later "Finished" entries, or
re-reads), each with its own session history. Up to 200 sessions are kept per reading entry; older
sessions are dropped on save.

## Generated index: `catalog/`

The `catalog/` directory is a denormalized, alphabetically-organized view of `books/`, built for fast
on-device browsing (see [USER_GUIDE.md](../USER_GUIDE.md#4-physical-books)). It is rebuilt automatically
whenever `/.myne/sync_needed` is present (created by the web API after any book/collection
create/update/delete/rename) — shown on-device as the "Rebuilding catalog…" screen. **It is always safe
to delete** — Myne will regenerate it from `books/` and `collections.ndjson` on the next rebuild.

### `catalog/idx.bin`

27 × `uint16_t` (little-endian), one count per letter section:

| Index | Section | Index | Section |
|---|---|---|---|
| 0 | A | ... | ... |
| 25 | Z | 26 | `#` (titles starting with a digit/symbol) |

Each value is the total number of entries (collections + standalone books) in that letter's
`.ndjson` file.

### `catalog/{A-Z,#}.ndjson`

One JSON object per line. **Collection headers** come first (sorted by name), followed by
**standalone books** (sorted by title) — both case-insensitive.

Collection header line:
```json
{"id":"a1b2c3d4","t":"Fantasy","c":1,"n":5,"e":7,"iv":1,"note":"optional note"}
```
`c: 1` marks this as a collection header; `n` is the number of books currently in the collection;
`e` is the expected total number of books (omitted when unset); `iv` is the initial volume number
(omitted when unset); `note` is included if the collection has a note set.

Book line (standalone, or as a header — collection membership is implied by which file it's in,
not stored per-line):
```json
{"id":"<book id>","t":"title","a":"author","l":"location","v":"volume"}
```

Field lengths are truncated for display: title/collection name ≤32 bytes, author ≤20 bytes,
location/volume ≤16 bytes, note ≤64 bytes. Max line length is 256 bytes.

### `catalog/c/{id8}.ndjson`

One line per book in the collection identified by `id8` (an 8-hex-character collection ID — see
below), using the same book-line schema as above, sorted by title. There is no header line in this
file — the collection's header lives in the relevant letter file.

## Persistent metadata (survives catalog rebuilds)

### `collections.ndjson`

The persistent registry mapping collection names to IDs, one per line:
```json
{"id":"a1b2c3d4","n":"Fantasy","e":7,"iv":1}
```
- `id` — an 8-character lowercase hex string, generated via `DataStoreUtils::generateHexId()` the
  first time a collection name is seen, and reused for that name afterwards (collision-checked
  against existing entries)
- `n` — the original collection name (up to 96 bytes UTF-8)
- `e` — optional expected total number of books in the collection; omitted or `0` means unset
- `iv` — optional initial volume number for the collection; omitted or `0` means unset

### `notes/collections/{id8}.note`

A per-collection note, stored as **plain UTF-8 text** (not JSON), up to 64 bytes. A missing or empty
file means no note is set. Written via the dashboard's collection note editor
(`POST /api/collections/note`).

## Caches (derived, can be regenerated)

### `readings-sum/{bookId}.bin` — 17 bytes

Fast summary of a book's reading status, used by the book detail screen so it doesn't need to parse
the full `readings/{bookId}.json`. Generated lazily the first time it's needed.

| Bytes | Field | Type | Notes |
|---|---|---|---|
| 0 | `hasReading` | `uint8_t` | `0` = no reading entries, `1` = has at least one |
| 1 | `status` | `uint8_t` | Reading status enum (0–4, see below) |
| 2 | `readingType` | `uint8_t` | `0` = Page, `1` = Chapter |
| 3–6 | `lastPosition` | `int32_t` LE | Position from the most recent session |
| 7–16 | `lastDate` | `char[10]` | `YYYY-MM-DD`, zero-padded |

The summarized reading entry is chosen by preferring one with status *Reading*, falling back to the
most recently updated entry.

### `stats-cache/summary.bin` — 40 bytes

Library-wide stats shown on the [Reading Stats](../USER_GUIDE.md#5-reading-stats-library-wide) Overview
screen.

| Bytes | Field | Type |
|---|---|---|
| 0–3 | `totalBooks` | `int32_t` LE |
| 4–7 | `totalReadings` | `int32_t` LE |
| 8–11 | `totalSessions` | `int32_t` LE |
| 12–15 | `byStatus[WantToRead]` | `int32_t` LE |
| 16–19 | `byStatus[Reading]` | `int32_t` LE |
| 20–23 | `byStatus[Paused]` | `int32_t` LE |
| 24–27 | `byStatus[Finished]` | `int32_t` LE |
| 28–31 | `byStatus[Dropped]` | `int32_t` LE |
| 32–33 | `byTracking[Page]` | `int16_t` LE |
| 34–35 | `byTracking[Chapter]` | `int16_t` LE |
| 36–37 | `latestYear` | `int16_t` LE |
| 38–39 | `latestMonth` | `int16_t` LE (1–12) |

### `stats-cache/{year}.bin` — 24 bytes

12 × `int16_t` (little-endian), one session count per month (January first), for the "Year" view.

### `stats-cache/{year}-{mm}.bin` — 62 bytes

31 × `int16_t` (little-endian), one session count per day of the month (day 1 first), for the "Month"
view. `mm` is zero-padded (`01`–`12`).

## Reading status & type enums

| Value | Status | API string |
|---|---|---|
| 0 | Want to Read | `want` |
| 1 | Reading | `reading` |
| 2 | Paused | `paused` |
| 3 | Finished | `finished` |
| 4 | Dropped | `dropped` |

| Value | Type |
|---|---|
| 0 | Page |
| 1 | Chapter |

## `sync_needed`

An empty flag file. Its presence tells Myne the `catalog/` directory is stale and must be rebuilt from
`books/` and `collections.ndjson` — shown on-device as the "Rebuilding catalog…" screen (see
[CatalogSyncActivity](../USER_GUIDE.md#2-powering-on)). The web API creates this flag after any change to
books, collections, or collection renames.
